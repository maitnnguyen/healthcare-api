# models/

This folder holds trained ML model files generated by `cancer-ml-models`.

## Files expected here (gitignored — generate locally)

| File | Generated by |
|---|---|
| `cancer_risk_model.pkl` | `cancer-ml-models/train_cancer_risk_model.py` |
| `cancer_risk_metadata.json` | same script |
| `cancer_risk_sample_data.csv` | same script |

## How to populate

```bash
# 1. Run training script in cancer-ml-models
cd ../cancer-ml-models
python train_cancer_risk_model.py

# 2. Copy outputs here
cp cancer_risk_model.pkl       ../healthcare-api/models/
cp cancer_risk_metadata.json   ../healthcare-api/models/
cp cancer_risk_sample_data.csv ../healthcare-api/models/
```

## Render deployment

On Render, model files are included at build time.
Add them before pushing your final deploy commit,
or use Render's persistent disk / environment to load them at runtime.
